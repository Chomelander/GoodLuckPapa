### 🐛 "我发现了 Bug"

**推荐路径**：
1. 打开浏览器开发者工具（F12），查看控制台和网络请求
2. 尝试用 `index.html`（原始主文件）重现问题
3. 查看 `js/db.js` 和相关 UI 文件理解逻辑
4. 提交 Issue 或联系维护者

**常见问题**：
- 数据丢失 → 检查 IndexedDB 是否被浏览器清空（私密浏览模式会导致）
- API 同步失败 → 检查后端是否启动，API 地址是否正确
- 老版本缓存 → 按 Ctrl+Shift+R 强制刷新清除 Service Worker

### 📊 "我想理解 App 的技术架构"

**推荐路径**：
1. 读 `README.md` 的「项目结构」部分
2. 查看 `js/db.js` — IndexedDB schema
3. 查看 `js/sync.js` — NAS 同步层
4. 查看 `js/app.js` — 应用初始化

**关键概念**：
- **IndexedDB**：浏览器本地数据库，8 个 store
  - `profile` — 孩子档案
  - `records` — 观察记录
  - `milestones` — 里程碑库
  - `milestoneStates` — 里程碑达成状态
  - `diaryEntries` — 育儿日记
  - `customActivities` — 自定义活动
  - `monthlyReviews` — 月度评估
  - `settings` — 设置（含 guideIntensity）

- **Service Worker**：离线缓存，版本号 `qiqi-v7`

- **sync.js**：Push-only 同步层，调用后端 REST API

- **NAS 后端**：Bun + SQLite，REST API 接口

## 常见问题

**Q: 我不会编程，只想用 App？**  
A: 打开 `index.html`，什么都不用装。PWA 形式，直接离线可用。

**Q: 我想把数据备份到云端？**  
A: 看 `部署文档/QUICK-START.md`，用 Docker 启动后端（5 分钟）。

**Q: App 中的「里程碑」是什么意思？**  
A: 查看 `分类方案/02_核心框架与模板.md` 的 D 部分，或打开 App 里程碑 Tab。

**Q: 活动库里的 「P-6-04」是什么意思？**  
A: 查看 `分类方案/02_核心框架与模板.md` 的 H 部分（编号体系）。P = 日常生活，6 = 6 月龄，04 = 第 4 号活动。

**Q: 我想为项目贡献代码？**  
A: 读 `CLAUDE.md` 的开发规范和技能分工，然后看 `部署文档/GIT-WORKFLOW.md`。

**Q: 测试全部通过吗？**  
A: 是的。运行 `npm test` 查看 244/244 通过。
