# 起起成长 P1 设计文档

## 背景

P0 已完整交付：纯前端 PWA（index.html），规则驱动，无服务端，243 测试全通。
P1 目标：引入 AI 功能 + 微信小程序版本。

## 范围

| 内容 | 说明 |
|------|------|
| PWA（现有） | 不变，继续维护，保留 IndexedDB 本地存储 |
| CF Workers 后端 | 新建，服务 PWA 和小程序双端 |
| 微信小程序 | 新建，uni-app Vue 3，完整 P0 功能 + P1 AI |

## 二、后端 API 设计

### 端点列表

| 端点 | 方法 | 鉴权 | 用途 |
|------|------|------|------|
| `/api/auth/login` | POST | 无 | 微信 code 换 JWT |
| `/api/profile` | GET / POST / PUT | JWT | 孩子档案 CRUD |
| `/api/records` | GET / POST | JWT | 观察记录读写 |
| `/api/records/:id` | PUT / DELETE | JWT | 记录编辑/删除 |
| `/api/milestones/states` | GET / PUT | JWT | 里程碑状态同步 |
| `/api/diary` | GET / POST / DELETE | JWT | 育儿日记 CRUD |
| `/api/ai/chat` | POST | 可选 JWT | AI 功能（周方案/里程碑分析/问答） |
| `/api/health` | GET | 无 | 健康检查 |

### 请求头约定（AI 端点）

```
Authorization: Bearer <JWT>         // 登录用户（小程序）
X-User-Key: sk-xxx                  // BYOK key（免费用户）
X-User-Type: free | subscriber      // 用户类型（P1 先 free）
X-Device-Id: <uuid>                 // 限流计数 key（PWA 端）
```

### AI 请求格式

```json
{
  "model": "gpt-4o-mini",
  "messages": [
    { "role": "system", "content": "<功能专属 system prompt>" },
    { "role": "user",   "content": "<结构化数据或用户输入>" }
  ],
  "stream": true
}
```

### AI 功能权限

| 功能 | 用户类型 | System Prompt 数据来源 |
|------|----------|----------------------|
| AI 周方案生成 | free + subscriber | 月龄 + 近7天记录 + 里程碑状态 |
| AI 里程碑分析 | free + subscriber | 月龄 + 里程碑达成情况 |
| AI 对话问答 | 仅 subscriber | 孩子档案 + 对话历史 |

## 三、数据库 Schema（CF D1）

```sql
-- 用户表
CREATE TABLE users (
  id         TEXT PRIMARY KEY,   -- WeChat openid
  created_at INTEGER NOT NULL,
  plan       TEXT DEFAULT 'free' -- free | subscriber
);

-- 孩子档案
CREATE TABLE profiles (
  id         TEXT PRIMARY KEY,
  user_id    TEXT NOT NULL REFERENCES users(id),
  name       TEXT,
  birth_date TEXT NOT NULL,
  gender     TEXT,
  created_at INTEGER NOT NULL
);

-- 观察记录
CREATE TABLE records (
  id          TEXT PRIMARY KEY,
  user_id     TEXT NOT NULL REFERENCES users(id),
  profile_id  TEXT NOT NULL REFERENCES profiles(id),
  activity_id TEXT NOT NULL,
  init_type   TEXT,
  focus_sec   INTEGER,
  emotion     TEXT,
  note        TEXT,
  created_at  INTEGER NOT NULL
);

-- 里程碑状态
CREATE TABLE milestone_states (
  id           TEXT PRIMARY KEY,
  user_id      TEXT NOT NULL REFERENCES users(id),
  profile_id   TEXT NOT NULL REFERENCES profiles(id),
  milestone_id TEXT NOT NULL,
  status       TEXT NOT NULL,    -- notStarted | emerging | achieved
  achieved_at  INTEGER
);

-- 育儿日记
CREATE TABLE diary_entries (
  id         TEXT PRIMARY KEY,
  user_id    TEXT NOT NULL REFERENCES users(id),
  profile_id TEXT NOT NULL REFERENCES profiles(id),
  content    TEXT,
  images     TEXT,              -- JSON array，CF R2 URLs
  created_at INTEGER NOT NULL
);

-- AI 使用量
CREATE TABLE ai_usage (
  user_id TEXT NOT NULL,
  date    TEXT NOT NULL,         -- YYYY-MM-DD
  count   INTEGER DEFAULT 0,
  PRIMARY KEY (user_id, date)
);
```

### 限流规则

| 用户类型 | 每日 AI 上限 |
|----------|------------|
| 免费用户 | 10 次/天 |
| 订阅用户 | 100 次/天 |

KV 条目 TTL 25 小时，自动过期。

### 错误码

| 场景 | HTTP | 前端提示 |
|------|------|---------|
| BYOK key 无效 | 401 | API Key 无效，请检查设置 |
| 超出每日限额 | 429 | 今日 AI 次数已用完，明天再试 |
| 功能需订阅 | 403 | 该功能需订阅后使用 |
| 上游超时 | 504 | AI 响应超时，请重试 |
| 内部错误 | 500 | 服务异常，请稍后再试 |

## 四、登录鉴权流程

```
小程序                    CF Workers                   微信服务器
  │                           │                            │
  │── wx.login() ────────────►│                            │
  │                           │── code2session ───────────►│
  │                           │◄── openid + session_key ───│
  │                           │  生成 JWT（含 openid，7天有效期）
  │◄── { token: <JWT> } ──────│                            │
  │  存 uni.setStorageSync     │                            │
  │                           │                            │
  │── 后续请求 ────────────────►│                            │
  │  Authorization: Bearer JWT │                            │
```

## 五、微信小程序（uni-app）

### 技术栈

| 层 | 选型 |
|----|------|
| 框架 | uni-app Vue 3 + Composition API |

## 六、开发阶段规划

| 阶段 | 内容 | 产出 |
|------|------|------|
| P1-a | CF Workers 骨架：D1 schema + auth + CRUD API | 可用后端 |
| P1-b | uni-app 初始化 + 登录 + 今日推荐 + 观察记录 | 小程序核心主路径 |
| P1-c | 里程碑 + 成长 Tab + 周/月复盘（完整 P0 移植） | 功能完整小程序 |
| P1-d | AI 功能接入（周方案 + 里程碑分析 + 问答） | P1 核心价值交付 |
| P1-e | 订阅鉴权（延后迭代） | 商业化基础 |
