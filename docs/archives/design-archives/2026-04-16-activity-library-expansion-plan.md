# 活动库扩充（0-6岁）实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将起起成长活动库从当前约 10 条扩充至覆盖 0-72 月（0-6 岁）的完整库，约 150 条活动，输出为单一文件 `activities-complete.js`。

**Architecture:** 三阶段执行——①按月龄段生成活动目录（Lin 逐段审核确认）→ ②批量填充完整 v1.5 格式 JS 数据 → ③集成验证。活动目录按 24 个 3 月子阶段组织，每段输出活动列表 + 维度覆盖矩阵，Lin 确认后方进入字段填充阶段。

**Tech Stack:** 原生 JS（无框架）、IndexedDB（活动数据在前端加载）、现有 v1.5 活动数据结构

**关键参考文件：**
- `分类方案/03_年龄段索引.md`：各段敏感期、WHO 节点、推荐活动方向（必读）
- `分类方案/04_示例活动集_0-1岁.md`：0-1 岁框架索引与维度覆盖矩阵
- `分类方案/05_示例活动_3-4岁.md`：3-4 岁标杆活动（倒水练习完整案例）
- `activities-batch-0-6mo.js`：现有 v1.5 格式参考（待废弃）
- `docs/plans/2026-04-16-activity-library-expansion-design.md`：本计划的设计依据

## 执行注意事项

1. **阶段一必须在阶段二之前全部完成**：所有 24 段目录经 Lin 确认后才开始写 JS 数据
2. **每段字段填充时参考格式**：严格对照 `activities-batch-0-6mo.js` 中的现有活动结构，字段不能少
3. **observeAnchor**：≤25 字，第一人称视角（"他…" 开头）
4. **theoryBite**：≤30 字 + 具体行为描述，口语化
5. **phases 的 range**：`[1, 2]` / `[3, 6]` / `[7, null]` 代表该活动使用次数区间，不是月龄
6. **linkedMilestones**：格式 `m_{domain}_{里程碑名}`，与 milestone-sample-phase1.js 中的 id 保持一致
7. **3-6 岁活动 minAge/maxAge**：使用月龄数值（如 36 月、48 月），不是年龄
