## 💾 数据库设计

### IndexedDB Schema（前端）
8 个 store，详见需求说明书：
- `profile` — 孩子档案
- `records` — 观察记录
- `milestones` — 里程碑库
- `milestoneStates` — 里程碑达成状态
- `diaryEntries` — 育儿日记
- `customActivities` — 自定义活动
- `monthlyReviews` — 月度评估
- `settings` — 设置（含 guideIntensity）

### NAS 后端 Schema（后端）
SQLite 数据库，详见需求说明书和 API 规范：
- `profiles` — 用户档案
- `records` — 观察记录
- `diary_entries` — 日记
- `milestones` — 里程碑定义
- `milestones/states` — 里程碑达成状态

## 🎯 设计原则

1. **极简主义** — 4 字段记录表，1 主 2 备推荐
2. **无评判** — 不奖励、不惩罚、不比较、不贴标签
3. **观察驱动** — 让父母看见孩子，而不是看见期待
4. **规则驱动** — MVP 无 AI，所有推荐都是可解释的规则
5. **离线优先** — 完全离线可用，云同步可选
