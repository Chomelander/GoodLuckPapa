# 起起成长 - AI 儿童发展顾问系统

## 一、产品概述

### 1.1 产品定位

基于多流派教育理论、依托 AI 能力的儿童发展顾问系统。为 0-6 岁孩子的父母（尤其是理工科背景）提供：

- 科学的发育评估与里程碑对照
- 个性化的成长方案制定与动态调整
- 系统性的家庭观察日志
- 孩子兴趣点的发现与培养路径
- 潜在发育风险的早期预警

**核心差异**：不是「育儿百科」，不是「打卡工具」，而是一个**持续学习你的孩子、不断优化建议的 AI 发展顾问**。

### 1.2 核心理念

- **科学底座**：以蒙特梭利、皮亚杰、多元智能等经过验证的教育理论为依据，不造概念、不贩卖焦虑
- **闭环进化**：观察 → 评估 → 方案 → 执行 → 再观察，AI 在每个环节提供智能支持
- **尊重个体**：每个孩子的发展节奏不同，系统基于「你的孩子」的实际数据给出建议，而非套用通用标准
- **低摩擦**：父母每天主动投入 < 5 分钟，其余融入日常陪伴

### 1.3 产品形态

| 端   | 形态       | 职责                              |
| --- | -------- | ------------------------------- |
| 手机端 | PWA / H5 | 日常使用：观察记录、方案执行、成长轨迹查看           |
| 服务端 | API 服务   | AI 能力：Prompt 拼装、模型调用、教育知识库、订阅鉴权 |

## 二、教育理念内核

> 这是整个系统的知识基础。所有功能模块——活动库、里程碑、AI 分析、健康预警——都建立在这套理论框架之上。

### 2.1 理论组合架构

系统不依赖单一教育流派，而是分层组合多个经过验证的理论：

```
┌─────────────────────────────────────────────┐
│          评估层：多元智能理论（Gardner）         │
│          定义发展维度、评估框架、个性化基础        │
├─────────────────────────────────────────────┤
│          方法论主框架：蒙特梭利（Montessori）    │
│          敏感期、月龄活动体系、有准备的环境、       │
│          观察记录方法                           │
├─────────┬───────────┬───────────────────────┤
│  0-1 岁  │  情感/社交  │     习惯/品格          │
│  补充：   │  补充：     │     补充：             │
│  RIE     │  瑞吉欧    │     正面管教            │
│          │  正面管教   │     华德福（节律）       │
├─────────┴───────────┴───────────────────────┤
│          科学底座：皮亚杰认知发展阶段理论          │
│          认知维度的分期标准和评估依据              │
└─────────────────────────────────────────────┘
```

### 2.2 各理论在系统中的角色

| 理论 | 角色 | 系统中的具体体现 |
|------|------|--------------|
| **蒙特梭利** | 方法论主框架 | 敏感期驱动的活动库；月龄精准对照；环境设计建议；观察记录方法论 |
| **皮亚杰** | 认知维度科学底座 | 感知运动期 6 个子阶段作为认知里程碑标准；认知活动难度分级依据 |
| **多元智能** | 评估与个性化框架 | 五维度模型的理论基础；发现孩子的优势智能组合；个性化方案的依据 |
| **RIE** | 0-1 岁核心补充 | 日常照料即教育；观察优先于干预；自由运动发展原则 |
| **正面管教** | 情感/习惯维度方法论 | 1.5 岁后的亲子沟通策略；规则建立方式；情绪引导话术 |
| **瑞吉欧** | 兴趣发掘方法论 | 从观察记录中发现兴趣 → 项目式深入探索；「一百种语言」激发多元表达 |
| **华德福** | 节律与自然参考 | 作息节律设计参考；自然材料偏好建议 |

### 2.3 理论如何驱动系统功能

#### 敏感期 × 活动库（蒙特梭利）

蒙氏定义的敏感期直接映射到系统的活动推荐引擎：

| 敏感期 | 年龄范围 | 系统行为 |
|--------|---------|---------|
| 秩序敏感期 | 0-3 岁 | 推荐固定流程的活动；提醒父母维持环境一致性 |
| 感官敏感期 | 0-5 岁 | 推荐多感官刺激活动；按月龄递进复杂度 |
| 语言敏感期 | 0-6 岁 | 语言维度活动密度根据阶段自动调整 |
| 运动敏感期 | 0-4 岁 | 大运动→精细动作递进；结合 RIE 的自由运动原则 |
| 社交敏感期 | 2.5-5 岁 | 触发社交类活动解锁；引入正面管教话术 |

#### 认知阶段 × 里程碑（皮亚杰）

皮亚杰感知运动期的 6 个子阶段作为 0-2 岁认知里程碑的科学标准：

| 子阶段 | 月龄 | 核心能力 | 系统里程碑 |
|--------|------|---------|-----------|
| 1. 反射活动 | 0-1m | 先天反射 | 吸吮反射、抓握反射 |
| 2. 初级循环反应 | 1-4m | 重复自身动作 | 重复吸手指、注视双手 |
| 3. 次级循环反应 | 4-8m | 重复引起外部效果的动作 | 摇铃铛、拍打玩具 |
| 4. 次级图式协调 | 8-12m | 目的性行为、客体永久性 | 寻找藏起的玩具、绕障取物 |
| 5. 三级循环反应 | 12-18m | 主动实验 | 从不同高度扔东西、用不同方式探索 |
| 6. 心理表征 | 18-24m | 符号思维 | 延迟模仿、假装游戏 |

#### 多元智能 × 个性化方案（Gardner）

系统追踪孩子在各智能维度的表现，生成「智能画像」，据此调整方案侧重：

```
系统持续观察数据 →
  发现：身体动觉智能 ★★★★★，音乐智能 ★★★★，语言智能 ★★★
    ↓
  AI 建议：
  · 运动类活动可提高难度，孩子明显有优势
  · 尝试引入节奏类活动（拍手歌、鼓点模仿）
  · 语言维度保持当前强度，不急于加码
  · 整体节奏：顺应孩子的优势智能，同时温和拓展其他维度
```

### 2.4 教育理念在 AI Prompt 中的体现

服务端的 System Prompt 需要内化这套理论框架：

```
你是一个专业的儿童发展顾问。你的知识体系基于以下教育理论的组合：

【主框架】蒙特梭利：
- 根据孩子当前月龄所处的敏感期推荐活动
- 活动设计遵循"有准备的环境"原则
- 优先让孩子自主探索，成人角色是观察者和环境设计者

【认知评估】皮亚杰：
- 根据感知运动期的子阶段评估认知发展水平
- 活动难度匹配当前认知阶段，略高于舒适区

【个性化】多元智能：
- 关注孩子在不同智能维度的表现差异
- 方案制定顺应优势智能，温和拓展弱势维度

【0-1岁特别原则】RIE：
- 日常照料（喂食、换尿布、洗澡）本身就是教育
- 尊重婴儿的自主运动发展节奏
- 观察和陪伴优先于刻意训练

【情感引导】正面管教（1.5岁+）：
- 和善而坚定的沟通方式
- 帮助孩子识别和表达情绪

【兴趣发掘】瑞吉欧：
- 从观察记录中识别孩子反复出现的兴趣信号
- 建议围绕兴趣点进行深度探索

重要原则：
- 不贩卖焦虑，发育有个体差异是正常的
- 给出建议时注明理论依据
- 发现异常时区分"正常个体差异"和"需要关注的信号"
```

## 三、发展维度框架

### 3.1 维度定义（基于多元智能理论映射）

| 维度 | 代码 | 理论来源 | 关注内容 |
|------|------|---------|---------|
| 认知/智识 | `cognitive` | 皮亚杰 + Gardner 逻辑数学/空间智能 | 因果认知、逻辑分类、空间感知、专注力、STEM 启蒙 |
| 语言 | `language` | Gardner 语言智能 + 蒙氏语言敏感期 | 词汇输入与输出、发音、理解指令、阅读前技能 |
| 运动 | `motor` | Gardner 身体动觉智能 + RIE 运动发展 | 大运动（爬/走/跑）、精细动作、身体协调 |
| 情感/社交 | `emotional` | Gardner 人际/内省智能 + 正面管教 | 安全依恋、情绪识别与表达、共情、社交技能 |
| 习惯/品格 | `habit` | 华德福节律观 + 正面管教 + 蒙氏秩序敏感期 | 作息规律、自律、规则意识、价值观萌芽 |

### 3.2 阶段划分（双线并行）

系统同时参照蒙氏敏感期和皮亚杰认知阶段进行分期：

| 阶段 | 年龄段 | 蒙氏视角 | 皮亚杰视角 | MVP 覆盖 |
|------|--------|---------|-----------|---------|
| Phase 0 | 0–6m | 感官敏感期高峰 | 反射→初级循环反应 | 是 |
| Phase 1 | 6–12m | 运动/秩序敏感期启动 | 次级循环反应→图式协调 | 是（当前重点） |
| Phase 2 | 1–2 岁 | 语言敏感期高峰 | 三级循环反应→心理表征 | 二期 |
| Phase 3 | 2–3 岁 | 社交敏感期启动 | 前运算期早期 | 二期 |
| Phase 4 | 3–6 岁 | 文化/数学敏感期 | 前运算期 | 远期 |

### 3.3 维度 × 阶段矩阵（Phase 0-1）

| | 认知 | 语言 | 运动 | 情感 | 习惯 |
|--|------|------|------|------|------|
| **0-3m** | 视觉追踪、对比色感知 | 区分人声与环境音 | 抬头、手掌张开 | 识别主要照料者 | 昼夜节律建立 |
| **3-6m** | 因果初步（摇=响）、物体恒存萌芽 | 发音模仿、转向声源 | 翻身、抓握 | 社交微笑、分离焦虑萌芽 | 喂食/睡眠规律 |
| **6-9m** | 客体永久性（A-not-B）、模仿学习 | 牙牙学语、对名字有反应 | 独坐、爬行启动 | 陌生人焦虑、情绪回应 | 睡眠独立过渡 |
| **9-12m** | 目的性行为、工具使用 | 理解简单指令、首个词 | 扶站、巡航 | 社会参照、指向分享 | 固定作息形成 |

## 四、核心功能模块

### 4.1 家庭观察日志

> 理论依据：蒙特梭利观察法 + 瑞吉欧记录文化（Documentation）

#### 功能描述

父母记录孩子日常行为观察，系统提供结构化引导，AI 从中提取发展信号。

#### 观察记录类型

| 类型 | 触发场景 | 记录内容 | AI 处理 |
|------|---------|---------|--------|
| **日常观察** | 随时 | 自由文本描述孩子行为 | 自动识别维度归属、提取关键信号 |
| **活动反馈** | 执行完计划活动 | 完成度 + 反应评分 + 备注 | 评估活动难度匹配度 |
| **里程碑事件** | 观察到新能力 | 具体行为描述 + 时间 | 匹配里程碑库、更新发育评估 |
| **异常观察** | 发现不寻常行为 | 详细描述 + 频率 + 持续时间 | 触发健康预警分析 |

#### 观察引导框架（蒙氏观察法简化版）

系统在记录页面提供观察提示，帮助父母提高观察质量：

```
观察时请关注：
· 孩子主动做了什么？（而非你让他做什么）
· 重复了几次？（重复=兴趣信号）
· 遇到困难时的反应？（放弃/尝试/求助）
· 情绪状态？
· 持续了多长时间？（专注力指标）
```

### 4.2 成长里程碑对照

> 理论依据：WHO 发育标准 + 皮亚杰认知阶段 + 蒙氏敏感期

#### 功能描述

系统内置完整的 0-6 岁发育里程碑库，三重校验标准：

```
里程碑校验 = WHO/CDC 发育标准（医学）
           + 皮亚杰认知阶段（认知科学）
           + 蒙氏敏感期窗口（教育实践）
```

#### 里程碑状态机

```
预期中（pending）
    ↓ 进入预期窗口
观察中（watching）  ← 系统推送观察提示
    ↓ 父母标记达成
已达成（achieved）  ← AI 评估是否需要调整计划
    
观察中 → 延迟关注（delayed）← 超出预期窗口未达成
                              ↓ AI 评估
                         正常个体差异 / 建议关注
```

#### 里程碑展示

按维度和月龄展示，直观对照孩子当前进度：

```
运动维度 · 6-12个月

  ✓ 独坐不需支撑     预期6-8m   达成于6m10d  ✓ 提前
  ✓ 扶物站立         预期8-12m  达成于6m0d   ✓ 明显提前
  ◐ 爬行            预期7-10m  观察中        → 当前月龄7m
  ○ 巡航（扶走）     预期9-13m  预期中
  ○ 独立行走         预期11-15m 预期中
```

### 4.3 AI 个性化方案

> 理论依据：蒙氏活动体系 + 多元智能个性化 + 皮亚杰最近发展区

#### 功能描述

AI 基于孩子的实际发展数据（而非仅月龄），生成个性化的周活动方案。

#### 方案生成逻辑

```
输入：
├── 孩子当前月龄
├── 各维度里程碑进度（是否超前/延迟）
├── 近期观察日志（兴趣信号、行为模式）
├── 近期活动执行数据（完成率、反应评分）
├── 父母状态反馈（精力水平）
└── 当前所处敏感期

AI 处理：
├── 匹配敏感期 → 确定本周各维度权重
├── 评估最近发展区 → 确定活动难度
├── 分析兴趣信号 → 融入孩子偏好的元素
├── 考虑父母状态 → 调整活动总量
└── 理论交叉验证 → 确保建议有据可依

输出：
├── 本周活动清单（每日 1-2 个，可嵌入日常场景）
├── 每个活动的理论依据说明
├── 本周重点观察方向
└── 调整说明（相比上周变了什么、为什么）
```

#### 活动卡片设计

每个活动是最小执行单元：

| 字段 | 说明 |
|------|------|
| 维度 | 所属发展维度 |
| 理论来源 | 基于哪个教育理论推荐 |
| 阶段 | 适用发育阶段 |
| 时长 | 5–15 分钟 |
| 场景 | 嵌入日常场景（洗澡/喂食/游戏时间等） |
| 操作说明 | 父母具体怎么做 |
| 观察指引 | 做完后观察什么 |
| 进阶/降级 | 太简单或太难时如何调整 |
| 科学依据 | 为什么推荐这个活动（一句话） |

### 4.4 兴趣发掘引擎

> 理论依据：瑞吉欧「一百种语言」+ 多元智能优势识别

#### 功能描述

AI 持续分析观察日志和活动反馈，识别孩子的兴趣信号，并建议深入探索方向。

#### 兴趣识别逻辑

```
信号采集：
· 活动反应评分持续高分的类型
· 观察日志中反复出现的行为模式
· 孩子主动发起的活动（非父母引导）
· 专注时间异常长的场景

模式识别（AI）：
· 跨多次观察的一致性
· 与多元智能类型的关联
· 排除因新鲜感导致的短期兴趣

输出：
· 兴趣画像更新："音乐节奏类活动持续高分，建议深入"
· 探索建议：具体的延伸活动
· 智能画像：各智能维度的相对强弱
```

#### 兴趣报告示例

```
🔍 兴趣发现（基于近30天数据）

强信号：
  · 节奏/音乐：拍手歌反应评分连续4周5分，
    会自发拍打桌面制造节奏（观察日志×3次）
    → 关联智能：音乐智能、身体动觉智能
    → 建议：引入简单打击乐器、节奏模仿游戏

中等信号：
  · 物理因果：反复按压发声玩具，对"倒下再立起"的游戏着迷
    → 关联智能：逻辑数学智能
    → 建议：提供更多因果明确的玩具

观察中：
  · 社交互动：对同龄孩子表现出兴趣但尚不稳定
    → 需要更多数据，保持观察
```

### 4.5 健康预警系统

> 理论依据：WHO/CDC 发育筛查标准 + 皮亚杰阶段常模

#### 功能描述

系统持续对照发育标准，当出现潜在风险信号时，向父母发出分级预警。

#### 重要声明

```
本系统的健康预警功能仅供参考，不构成医学诊断。
任何预警建议中都会提示：如有疑虑，请咨询儿科医生或发育行为专家。
```

#### 预警触发条件

| 预警级别 | 触发条件 | 系统行为 |
|---------|---------|---------|
| **信息提示**（绿色） | 某里程碑接近预期窗口上限，尚未达成 | 推送观察提示 + 促进活动 |
| **关注建议**（黄色） | 某里程碑超出预期窗口未达成；或单一维度持续低活跃 | 详细分析 + 建议增加相关活动 + 建议关注 |
| **就医建议**（红色） | 多个关联里程碑延迟；或出现发育倒退信号 | 明确建议咨询专业医生 + 提供就医准备清单 |

#### 里程碑数据源（三源融合）

系统里程碑的「预期窗口」不依赖单一标准，而是融合三个权威数据源：

| 发展维度 | 主数据源 | 校准数据源 | 理由 |
|---------|---------|-----------|------|
| 运动 | WHO 大运动发育标准 | 国内卫健委 | WHO 全球多国大样本最权威，卫健委做中国适配 |
| 认知 | 皮亚杰阶段理论 | CDC 2022 修订版 | 皮亚杰提供理论框架，CDC 提供窗口参考 |
| 语言 | CDC 2022 修订版 | 卫健委 | CDC 2022 版覆盖最全，卫健委补充本土语言标准 |
| 社交/情感 | CDC 2022 修订版 | 卫健委 | 同上 |
| 习惯 | CDC + 蒙氏秩序敏感期 | 卫健委 | CDC 提供基线，蒙氏补充秩序/节律维度 |

**预期窗口取值规则**：
```
正常窗口 = 三源中最宽的窗口（宁可漏报，不制造焦虑）
```

**预警阈值梯度**：
```
信息提示（绿色）= 接近最宽窗口上限，尚未达成
关注建议（黄色）= 超出最宽窗口 + 1 个月
就医建议（红色）= 超出最宽窗口 + 2 个月，或出现发育倒退信号
```

#### 预警分析逻辑

```
AI 分析维度：

1. 里程碑延迟评估
   · 单一里程碑延迟 → 通常是个体差异，信息提示
   · 同维度多个里程碑延迟 → 关注建议
   · 跨维度关联延迟 → 就医建议

2. 发育倒退检测
   · 已达成的能力消失或明显退步 → 高优先级预警

3. 行为模式异常
   · 从观察日志中识别：
     - 持续回避眼神接触
     - 对声音无反应
     - 重复刻板行为
     - 感官过度敏感/迟钝

4. 个体差异校准
   · 综合考虑早产（矫正月龄）、家族发育节奏等因素
   · 区分"慢但正常"和"需要干预"
```

#### 预警输出示例

```
⚠️ 关注建议（黄色）

内容：语言维度里程碑"对名字有反应"（预期窗口 5-9m）
      目前 9m15d 仍未观察到稳定反应

AI 分析：
  · 运动维度和认知维度发展正常，排除整体发育延迟
  · 近期观察日志显示对环境声音有反应，排除听力问题可能性低
  · 可能原因：个体差异、日常呼唤方式、环境噪音

建议：
  1. 增加面对面近距离呼名练习，每天 5 次
  2. 排除环境因素：安静环境下测试呼名反应
  3. 如 2 周内无改善，建议咨询儿科医生

声明：以上分析仅供参考，不构成医学诊断。
```

### 5.2 闭环数据流

```
① 父母记录观察日志（手机本地）
        ↓
② 积累一周数据 → 发送摘要给服务端
        ↓
③ 服务端拼装 Prompt（用户数据 + 教育理论框架 + 活动库）
        ↓
④ AI 返回：周方案 + 发育评估 + 兴趣信号 + 预警检查
        ↓
⑤ 手机存储方案 → 父母执行 → 记录 → 循环
```


## 九、隐私与安全

### 9.1 原则

| 原则 | 措施 |
|------|------|
| 最小数据传输 | 每次 AI 请求只发送当次分析所需的摘要数据 |
| 服务端不持久化 | 用户育儿数据不写入服务端数据库和日志 |
| 传输安全 | 全链路 HTTPS |
| 透明可审计 | 用户可查看每次发送给 AI 的数据内容 |
| 用户可控 | 支持数据导出、一键清除 |

### 9.2 服务端数据边界

| 服务端存储 | 服务端不存储 |
|-----------|------------|
| 用户账户信息 | 孩子成长数据 |
| 订阅状态 | 观察日志内容 |
| API 调用计数 | 里程碑记录 |
| Prompt 模板 | AI 分析结果 |
| 活动库内容 | 家庭相关信息 |

## 十二、MVP 范围与开发计划

### 12.0 分阶段策略

```
阶段零（个人版）：Lin 自用，验证 AI 育儿闭环是否真的有用
阶段一（对外版）：加入注册/登录/付费，向其他家长开放
```

**阶段零不需要**：注册/登录、订阅/付费、多用户隔离、服务端

### 12.1 MVP 功能边界（阶段零·个人版）

**包含**：
- 孩子档案创建（首次启动引导）
- AI 周方案生成（Claude API 直连，API Key 本地存储）
- 今日活动卡展示与执行
- 活动打卡（完成/跳过 + 评分 + 备注）
- 观察日志记录（带引导提示 + 备注升级为观察）
- 里程碑对照与标记
- AI 里程碑评估（标记达成后触发）

**暂不包含（阶段零）**：
- 注册/登录/用户系统
- 订阅/付费
- AI 月度报告
- AI 兴趣发掘报告
- AI 健康预警
- 成长轨迹可视化（雷达图/趋势图）
- 语音输入 / PDF 导出
- PC 端 / 多孩子 / 家庭协作

### 12.2 技术架构（阶段零）

```
纯前端 PWA（无服务端）

手机浏览器
  ├── Vue 3 + Vite + PWA
  ├── Dexie.js（IndexedDB 封装）← 所有数据本地存储
  └── 直连 Claude API（API Key 存 localStorage）
```

### 12.3 开发方式

使用 **Claude Code Vibe Coding**：Lin 描述意图 → Claude Code 生成代码 → Lin 验收调整。

每次 Session 开头提供固定上下文包：

```
项目：起起成长 · AI儿童发展顾问 · 个人版（阶段零）
技术栈：Vue 3 + Vite + PWA + Dexie.js，无服务端，API Key 本地存储直连 Claude API
设计参考：prototype.html（UI 风格/颜色/组件以此为准）
本次任务：[具体描述]
```

### 12.4 开发节奏（预计 2 周 · 约 13 小时）

**第一周：界面与数据层**

| Session | 任务 | 预计时长 |
|---------|------|---------|
| S1 | 项目初始化：Vue3 + Vite + PWA + Dexie.js，底部导航壳子 | 1h |

| S3 | 今日活动卡：读取周方案渲染列表，打卡交互（完成/跳过/评分/备注） | 2h |
| S4 | 观察日志：记录表单 + 维度标签 + 备注升级观察逻辑 | 1h |
| S5 | 里程碑对照：按维度展示列表，状态标记 | 1h |

**第二周：AI 接入，跑通闭环**

| Session | 任务 | 预计时长 |
|---------|------|---------|
